Theoretical part

  • IP address

    It's a number given to a device using the IP protocol, which serves as an identifier in the third (network) layer of the OSI model. IP address can be assigned using the Address Resolution Protocol used to associate IP with MAC. It identifies the network interface of the host and provides the location of the host in the network, which means that a path to the device can be established. An IP address is generally used in order to find a device in a network.
    There exist two types of IP addresses: IPv4 and IPv6. The latter was defined, because there weren't enough IPv4 addresses.

    An IPv4 address has a size of 32 bits and is usually represented in dot-decimal notation (4 decimal numbers from 0 to 255 separated by dots). Exemplary IPv4 address source

    An IPv6 address is composed of 128 bits, so it greatly increases the amount of possible addresses (up to 2128) Exemplary IPv6 address source

  • Network subnets

    They are logical subdivisions of an IP network so that hosts in the same network share identical beginning of their IP addresses. The process of dividing a network into subnets is called subnetting. Subnetting process source
    This results in the logical division of an IP address into two fields, the network number or routing prefix and the rest field or host identifier. The rest field is an identifier for a specific host or network interface. For IPv4, a network may also be characterized by its subnet mask or netmask. They are also expressed in dot-decimal notation like an address. For example, 255.255.255.0 is the subnet mask for the prefix 198.51.100.0/24.
    Based on the subnet mask we can derive e.g. the broadcast address. Subnetting may also enhance routing efficiency, or have advantages in network management when subnetworks are administratively controlled by different entities in a larger organization.

    When several devices are in the same subnetwork, they can communicate without the use of routers. The hosts are identified by their physical addresses.

  • Network classes

    A classful network is a network addressing architecture used in the Internet from 1981 until the introduction of Classless Inter-Domain Routing in 1993. The method divides the IP address space for Internet Protocol version 4 (IPv4) into five address classes based on the leading four address bits. Classes A, B, and C provide unicast addresses for networks of three different network sizes. This architecture extended the addressing capability in the Internet, but did not prevent IP address exhaustion. The problem was that many sites needed larger address blocks than a Class C network provided, and therefore they received a Class B block, which was in most cases much larger than required. In the rapid growth of the Internet, the pool of unassigned Class B addresses was rapidly being depleted. source

    Class A comprises networks 1.0.0.0 through 127.0.0.0. The network number is contained in the first octet. It's mask is /8.

    Class B contains networks 128.0.0.0 through 191.255.0.0; the network number is in the first two octets. It's mask is /16.

    Class C networks range from 192.0.0.0 through 223.255.255.0, with the network number being contained in the first three octets. It's mask is /24.

    Classes D (Multicast, that is between groups), E (Experimental), and F (for future use). Addresses range from 224.0.0.0 through 254.0.0.0.

  • Network mask

    Subnet masks were introduced in response to the ineffective Classful routing (described in the previous point). It allows for grouping big networks into smaller subnetworks where hosts can be more easily identified.

    It is used to distinguish between the subnet address and the host address in an IP address. The mask address must be known to the routers and devices belonging to a subnet. When the router compares a mask address (e.g. 255.255.255.0) with an IP address of a host, it gets the information about which part of the address identifies the host, and which one identifies the subnet.

    For example for the subnet mask of 255.255.255.0 and the IP address of 192.168.1.25 we can deduce that the host identifier is 25, as it is comprised of the bits not "covered" by the mask.

    source

  • Routing table

    A routing table, is a set of data collected by a router, which contains routes to network destinations and sometimes their distances (metrics) as well. It has the knowledge of the immediate network topology.

    Data transfer between nodes in a network depends on correct routing of the packets. If a packet cannot be send directly, then a node will pass the IP packet to a gateway in its LAN in order for it to correctly establish a path to the correct destination. The routing table keeps these paths and uses them to determine the correct ones for data packages.

    Routing tables can also contain "next hop" information. They are basically another routers on the packet's way to its final destination. This is called hop-by-hop routing and is the characteristic of the network layer in the OSI model.

    There exist directly and remotely connected networks. The latter can be reached only by forwarding a data packet to another router through dynamic or static, manually configured routes.

    A routing table consists of at least 3 fields:
    Network ID - destination host subnet
    Metric - Used to determine the shortest path to the gateway
    Next hop - next router along the way to the packet's destination
